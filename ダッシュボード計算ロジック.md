# ダッシュボード計算ロジック仕様書

## データソース

### Firestoreコレクション
- **progressDashboard**: 全案件ドキュメント
  - `isExistingProject === true` → 既存案件
  - `isExistingProject !== true` → 新規案件
- **progressDashboard/{dealId}/salesRecords**: 各案件の営業記録（サブコレクション）

### salesRecordsの主要フィールド
| フィールド | 型 | 説明 |
|---|---|---|
| `recordType` | string | `"新規"` or `"継続"` |
| `budget` | number/string | 予算金額 |
| `date` | string (YYYY-MM-DD) | レコード登録日 |
| `phase` | string | フェーズ（フェーズ1〜8, Dead, 失注） |

### 案件ドキュメントの主要フィールド（パイプライン用）
| フィールド | 型 | 説明 |
|---|---|---|
| `status` | string | 現在のフェーズ |
| `expectedBudget` | number | 想定予算 |
| `representative` | string | 営業担当者 |
| `proposalMenu` | string | 提案メニュー |
| `leadSource` | string | 流入経路 |
| `confirmedDate` | string | 成約日 |
| `createdAt` | timestamp | 案件登録日 |

### フェーズ別受注確率（PHASE_PROBABILITY）
| フェーズ | 確率 |
|---|---|
| フェーズ1 | 5% |
| フェーズ2 | 15% |
| フェーズ3 | 25% |
| フェーズ4 | 50% |
| フェーズ5 | 70% |
| フェーズ6 | 90% |
| フェーズ7 | 95% |
| フェーズ8 | 100% |
| 失注 | 0% |

### 担当者リスト
- 取得元: Firestore `staffMembers` コレクション（`role === 'sales'`）
- フォールバック: データ内の`representative`フィールドから自動導出

## 新規案件ダッシュボード（NewDealsDashboard.js）

### データ取得（fetchData）
1. `progressDashboard` 全件取得
2. `isExistingProject !== true` → **newDeals**（パイプライン表示用）
3. `isExistingProject === true` → **existingDeals**（売上集計のベース）
4. existingDeals の各案件から `salesRecords` サブコレクションを並列取得 → **allSalesRecords**

### 1. チーム全体四半期売上実績
- **UI表示**: 目標達成率メーター（`quarterActual` / `quarterTarget`）
- **state**: `quarterActual`
- **ロジック**:
  - 対象: `allSalesRecords` のうち `recordType === '新規'` かつ `date` が今四半期内
  - 計算: 対象レコードの `budget` を合計

### 2. 四半期内月別売上実績
- **UI表示**: 棒グラフ（3ヶ月分）
- **state**: `quarterMonthlyActual`
- **ロジック**:
  - 四半期内の各月ごとに:
    - 対象: `allSalesRecords` のうち `recordType === '新規'` かつ `date` がその月内
    - 計算: 対象レコードの `budget` を合計

### 3. チーム全体四半期売上見込み（担当者別）
- **UI表示**: 円グラフ（担当者別）
- **state**: `quarterForecast`
- **ロジック（2パート合算）**:
  - **パートA（パイプライン）**: `newDeals`（新規案件一覧）のうち `status` がフェーズ1〜7（失注・Dead・フェーズ8除外）
    - 計算: `expectedBudget × PHASE_PROBABILITY[status]`
    - `representative` でグルーピング
  - **パートB（実績）**: `allSalesRecords` のうち `recordType === '新規'` かつ `date` が今四半期内
    - 計算: `budget` をそのまま加算
    - `representative` でグルーピング
  - 合計: パートA + パートB

### 4. チーム全体月間売上見込み（担当者別）
- **UI表示**: 円グラフ（担当者別）
- **state**: `monthForecast`
- **ロジック**: #3 と同一構造だが期間が「今月」
  - パートA: 同じ（フェーズ1-7は期間に依存しない）
  - パートB: `date` が今月内のレコード

### 5. チーム全体月間売上実績
- **state**: `monthActual`
- **ロジック**:
  - 対象: `allSalesRecords` のうち `recordType === '新規'` かつ `date` が今月内
  - 計算: 対象レコードの `budget` を合計

### 6. 個人四半期売上（担当者別テーブル）
- **UI表示**: テーブル（担当者名、金額）
- **state**: `quarterlyPersonalSales`
- **ロジック**:
  - 対象: `allSalesRecords` のうち `recordType === '新規'` かつ `date` が今四半期内
  - 計算: `budget` を `representative` でグルーピングして合計

### 7. 個人月間売上（担当者別テーブル）
- **UI表示**: テーブル（担当者名、金額）
- **state**: `monthlyPersonalSales`
- **ロジック**:
  - 対象: `allSalesRecords` のうち `recordType === '新規'` かつ `date` が今月内
  - 計算: `budget` を `representative` でグルーピングして合計

### 8. 担当者別案件サマリー（フェーズ2〜7）
- **UI表示**: 逆ピラミッド型ファネル（件数・予算）＋案件一覧テーブル
- **state**: `representativeSummary`
- **ロジック**:
  - 対象: `newDeals` のうち `representative === 選択中の担当者` かつ `status` がフェーズ2〜7
  - 件数: フェーズごとにカウント
  - 予算: フェーズごとに `expectedBudget` を合計
  - 一覧: 会社名、フェーズ、想定予算を表示（フェーズ7→2の順）

### 9. 滞留商談リスト（90日以上）
- **UI表示**: テーブル（会社名、提案メニュー、経過日数、予算）
- **state**: `stagnantDeals`
- **ロジック**:
  - 対象: `allSalesRecords` のうち `phase` がフェーズ8・失注・Dead以外
  - 条件: `date` から現在まで90日以上経過
  - 表示: 会社名、提案メニュー、経過日数、`budget`

### 10. サービスごとの流入経路（売上ベース・期間指定付き）
- **UI表示**: サービス別の円グラフ＋日付範囲ピッカー
- **state**: `leadSourceData`（useMemo）, `leadSourceDateFrom`, `leadSourceDateTo`
- **ロジック**:
  - 期間: カスタム日付指定時はその範囲、未指定時は今四半期
  - 対象: `allSalesRecords` のうち `recordType === '新規'` かつ `date` が期間内
  - グルーピング: `proposalMenu`（サービス） × `leadSource`（流入経路）
  - 計算: `budget` を合計

## 新規と既存の違い一覧

| 項目 | 新規ダッシュボード | 既存ダッシュボード |
|---|---|---|
| salesRecords フィルタ | `recordType === '新規'` | `recordType === '継続'` |
| 売上見込みパートA | `newDeals`のフェーズ1-7 | `existingDeals`のフェーズ1-7 |
| 担当者別サマリー | `newDeals`のdeal.statusベース | `allSalesRecords`のrec.phaseベース |
| 流入経路セクション | あり（期間指定付き） | なし（削除済み） |
| メニュー別/会社別 | 削除済み | 削除済み |
